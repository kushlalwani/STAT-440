Exercise 1

This question is taken directly from the midterm exam for STAT 440 in FA 2023. The purpose of this question is to give you some practice with the style of questions you can expect on an exam.

Let \(X_1, \ldots, X_n\) be an i.i.d.random sample where each \(X_i\) has the density function \(g(x)= 4\exp\{-4(x-5)\} I\{x>5\}\).

Now suppose that \(Z\) denotes a standard normal random variable. [The standard normal density function was provided on the front page of the midterm exam in FA 2023.] We wish to approximate the probability \(P(Z > 5)\) using importance sampling. This probability may be expressed as follows: \[ P(Z > 5) = \int_5^\infty \phi(x)\,dx. \] The importance sampling approximation should take the form of a sample mean, \((1/n) \sum_{i=1}^n s(X_i)\), for some function \(s(X_i)\).

Derive the correct form of \(s(X_i)\) and explain your work.

Answer

First we start with the density function of the standard normal model, which is the distribution from which we are trying to sample from: \(F_X(x) = \frac1{\sqrt{2\pi}}e^{- \frac{x^22}\)

Then we can apply importance sampling, which states that we can multiply and divide by another density function that is easy to sample from distribution that is also heavier tailed than the standard normal model. If we do not select a distribution that is heavier tailed than the standard normal, it could lead to infinite variance in our approximation. We know that our function g(x) is heavier tailed than the normal model because the exponential part of g(x) has a linear x-term instead of the quadratic x-term in the normal model, so the normal model will decay faster than g.

We are given that \(P(X>5)=\int_5^\infty \phi(x)dx \approx \frac1n\sum_{i=1}^n s(X_i)\). So we need to derive \(s(X_i)\)

From Importance Sampling, we know that: \(I = \int \phi(x)dx = \int\phi(x)\frac{g(x)}{g(x)}dx = E_g[\frac{\phi(x)}{g(x)}] \approx \frac1n\sum_{i=1}^n\frac{\phi(x)}{g(x)}\)

Now we get that \(s(X_i) = \frac{\phi(x)}{g(x)}\), and we can just plug in out density functions in.

\(s(x) = \frac{\frac1{\sqrt{2\pi}}e^{-\frac{x^2}{2}}}{4e^{-4(x-5)} I_{\{x>5\}}}I_{\{x>5\}} = \frac1{4\sqrt{2\pi}}e^{-\frac{x^2}{2}+4x-20}\)

\(s(X_i)=\frac1{4\sqrt{2\pi}}e^{-\frac{X_i^2}{2}+4X_i-20}\)

Answer

Part (a)

lmean = function(lx) { #copy pasted from the notes
m = max(lx)
return(m + log(mean(exp(lx-m))))
}

Now we will attempt to use the new function to find P(Z>100)

set.seed(440)

X <-rnorm(1e4,mean=100)

lmean((X>100) * log(dnorm(X) / dnorm(X-100)))
## [1] NaN

Running this code results in Not a Number. This is because the number that we are trying to compute is extremely small, so the computer cannot handle it.

Importance sampling says that \(P(X>100) = E_g[I_{\{x>100\}}\log(\frac{\phi(x)}{g(x)})]\) in our code above, \(\phi(x)\) is the density function for the standard normal model and \(g(x)\) is the density function for \(N(100,1)\). After plugging both functions in, we will get: \(E_g[I_{\{x>100\}}\frac{\phi(x)}{g(x)}] = E_g[I_{\{x>100\}} \log(e^{\frac{200x-10000}2}) ] = E_g[I_{\{x>100\}}\frac{200x-10000}2]\)

As we can see, for any values of x that are greater than 100, the values of x become extremely large. When computing the log mean from the function, we are exponentiating the already large number. This will result in an even larger number that the computer cant handle, resulting in the program throwing the NaN error. Subtracting the max as the function does, wouldnt help either. This is because an extremely large value for lx could result in a negative number of large magnitude to be in the exp function. This will result in an extremely small number for which R will throw NaN as well.

Part (b)

To solve this issue, we can cut out a middle step which is taking the log of the proportion of the normal distributions. I part a, we found what the fraction part of the importance sample reduces to. So instead of doing complex logs, fractions, and other computationally expensive procedures, we can just plug the weights of the proportion into the lmean function.

weights <- (X > 100) * ((200 * X - 10000) / 2)
lmean(weights)
## [1] 5337.967

This solution avoid taking the log of the proportion of the densities, and then further exponentiation and another log in the lmean function.

Part (c)

set.seed(440)

X <- 100 + rexp(1e4)
lmean(log((X>100) * dnorm(X) / dexp(X-100)))
## [1] NaN

This code results in NaN as well. So we need to expand the importance sample and reduce to just a weight like we did in the previous part.

\(X \sim \mbox{expo}(\lambda = 1) + 100\) so then \(g(x) = e^{-(x-100)}I_{\{x>100\}}\)

\(\frac{\phi(x)}{g(x)} = \frac{\frac1{\sqrt{2\pi}}e^{-\frac{x^2}{2}}}{e^{-(x-100)}I_{\{x>100\}}} = \frac1{2\pi I_{\{x>100\}}}e^{\frac{-x^2+2x-200}2}\)

Then we will plug back into the importance sample: \(E_g[I_{\{x>100\}}\frac{\phi(x)}{g(x)}] = E_g[\frac1{2\pi}e^{\frac{-x^2+202x-10200}2}]\)

But since we are trying to find the log probability, we take the log of the proportion: \(\log(\frac1{2\pi}e^{\frac{-x^2+2x-200}2}) = \frac{-x^2+2x-200}2 - \log(2\pi)\)

Then we can use this as the weight for our importance sample.

set.seed(440)

weights <- (X > 100) * (-X^2/2 + X - 100 - log(sqrt(2*pi)))

lmean(weights)
## [1] -5005.33

Answer

According to the notes, the Antithetic Variates method for approximating an intergral is given by: \(\frac1n \sum^n_{i=1}\frac12 [h(U_i)h(1-U_i)]\)

set.seed(440)
reps <- 750
n <- 1500

h <- function(x){
  1/(1+x)
}

simple <- 1:reps
antithetic <- 1:reps

for (i in 1:reps) {
  U <- runif(2*n)
  simple[i] <- mean(h(U))
  V <- U[1:n]
  antithetic[i] <- mean((h(V) + h(1-V))/2)
}

resultTable <- data.frame(
  Method = c("Simple", "Antithetic"),
  MeanOfEstimates = c(mean(simple), mean(antithetic)),
  SE = c(sd(simple), sd(antithetic)))

kable(resultTable)

Antithetic 0.6930987 0.0006398
